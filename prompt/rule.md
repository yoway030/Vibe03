# 규칙 정의
에이전트 = AI 에이전트를 의미
진행자 = 에이전트를 사용하는 사람을 의미

# 최종 목표
- 최대 100인이 동시에 진행하는 멀티 온라인 땅따먹기 게임.

# 사용 기술
- C#
- Box2D : https://github.com/ikpil/Box2D.NET 참고

# 프로젝트 구조
```
/
├── app/                 # 제품 소스코드 디렉토리
│   ├── Module_1         # 일부 기능을 구현한 모듈 프로젝트 디렉토리
│   │   ├── Module_1                # 모듈 자체 코드
│   │   │   └── Module_1.csproj
│   │   ├── Module_1_test           # 모듈 유닛 테스트 코드. 세부 기능별로 하나하나 테스트 할 수 있도록 유닛 테스트를 추가한다.
│   │   │   └── Module_1.csproj
│   │   ├── Module_1_Sample         # 모듈 샘플 코드. 눈으로 확인할 수 있도록 그래픽 기능을 넣어 테스트 할수 있도록 한다.
│   │   │   └── Module_1.csproj
│   │   ├── Module_1_plan.md        # 모듈의 개발계획. 진행자가 작성. 에이전트가 해당 모듈 개발을 위해서 참조
│   │   ├── Module_1_progress.md    # 모듈의 개발 기록. 에이전트가 해당 모듈 개발 과정을 기록
│   │   ├── readme.md               # 모듈에 대한 문서화. 에이전트가 작성.
│   │   └── Module_1.sln            # 모듈과 샘플에 대한 솔루션 파일
│   ├── Module_...       # 모듈들을 의미
│   ├── app.sln             # 제품을 빌드하는 솔루션.
│   ├── app_progress.md     # 제품에 대한 작업 진행 상황 기록. 에이전트가 이전 기록을 참고 하고 작업을 진행하며 업데이트 한다
│   ├── app_rule.md         # 제품 소스 코드 작업에 대한 규칙
├── prompt/              # 프롬프트 설정
│   ├── github_config.md    # github 정보
│   ├── rule.md         # 현재 파일
│   └── ...
└── output/              # App의 최종 빌드 후 아웃풋 위치
    └── logs/            # 실행파일 실행시 로그가 출력되는 위치
```

# 참조 프롬프트
아래에 프롬프트 경로와 그 프롬프트의 역할을 설명한다.

## prompt/rule.md
에이전트는 수정하지 않는다.
현재 이 파일.
제품 개발을 위한 전체적인 구조, 빌드 방법, 사용 기술에 대한 규칙을 정의한다.

## prompt/github_config.md
에이전트는 수정하지 않는다.
개발 진행 내용을 저장하기 위한 정보
해당 내용을 참고하여 저장 할 만한 중간 변경 사항을 저장한다.
